Deploy Your Music Streaming App

GitHub → Netlify (frontend + API) → Cloudflare R2 (audio files)

Free tier friendly ~30 minutes
1
Put your code on GitHub
Netlify deploys directly from a GitHub repo
Step 1

Create a GitHub account (if you don't have one)

Go to github.com and sign up — it's free.

Step 2

Create a new repository

Click the + icon in the top-right → New repository.
Give it a name (e.g. my-music-app), set it to Public or Private — either works — then click Create repository.

Step 3

Push your project to that repo

In your project folder, run:

git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/my-music-app.git
git push -u origin main
First time? GitHub will ask you to sign in. Use your username and a Personal Access Token (GitHub → Settings → Developer settings → Personal access tokens) instead of your password.
2
Set up Cloudflare R2 for audio storage
Free egress — no charge when listeners stream your files
Step 4

Create a Cloudflare account

Go to cloudflare.com and sign up. You do not need to add a domain or credit card for R2's free tier.

Step 5

Create an R2 bucket

In the Cloudflare dashboard, click R2 in the left sidebar → Create bucket.
Name it something like music-uploads. Leave all other settings as-is and click Create bucket.

Step 6

Make the bucket publicly readable

Open your new bucket → click the Settings tab → scroll to Public access → click Allow Access.
Cloudflare will show you your public URL — copy and save it. It looks like:

https://pub-xxxxxxxxxxxxxxxx.r2.dev
This is your R2_PUBLIC_BASE_URL. Every file you upload will be available at YOUR_PUBLIC_URL/filename.mp3.
Step 7

Generate an API token

Still in R2 → click Manage R2 API tokens (top-right of the R2 page) → Create API token.
Set permissions to Object Read & Write for your bucket only. Click Create API token.

You'll see three values — copy them now (you won't see the secret again):

Value shown on screenThis becomes your env var
Account IDshown at the top of any R2 page
Access Key IDR2_ACCESS_KEY_ID
Secret Access KeyR2_SECRET_ACCESS_KEY

Your Account ID is on the right-hand sidebar of any R2 page — copy that too.

3
Deploy to Netlify
Netlify hosts the frontend and runs the Node API via serverless functions
Step 8

Create a Netlify account

Go to netlify.com → Sign up → choose Sign up with GitHub. This connects your account in one click.

Step 9

Import your GitHub repo

In the Netlify dashboard click Add new site → Import an existing project → Deploy with GitHub.
Authorize Netlify, then pick your my-music-app repository from the list.

Step 10

Configure the build settings

Netlify will auto-detect settings from netlify.toml in the repo. You should see:

# netlify.toml already in your project
[build]
  command  = "npm install"
  publish  = "public"

[[redirects]]
  from   = "/api/*"
  to     = "/.netlify/functions/api/:splat"
  status = 200

If the file already looks like this, leave everything as-is and continue.

Step 11

Add your environment variables

Before clicking deploy, click Add environment variables and add the following. Use the values you copied in Steps 6 and 7:

Variable nameWhere to find the value
R2_ACCOUNT_IDRight sidebar on any R2 page in Cloudflare
R2_ACCESS_KEY_IDCopied when you created the API token
R2_SECRET_ACCESS_KEYCopied when you created the API token
R2_BUCKET_NAMEThe bucket name you chose, e.g. music-uploads
R2_PUBLIC_BASE_URLThe https://pub-xxx.r2.dev URL from Step 6
JWT_SECRETAny long random string — e.g. run openssl rand -hex 32 in a terminal
Never put these values in your code or commit them to GitHub. Netlify's environment variables panel is the right place for secrets.
Step 12

Deploy

Click Deploy site. Netlify will build and deploy — this takes about 60 seconds.
When it finishes, you'll get a live URL like https://your-app-name.netlify.app.

Every future push to your GitHub main branch automatically redeploys. You never need to touch Netlify again for routine updates.
4
Verify everything works
Quick checklist before you share the link
  • Open your Netlify URL — the app loads without a blank screen
  • Register a new account through the app's UI
  • Upload a song — the upload completes without an error
  • Go to your R2 bucket in Cloudflare and confirm the file appears there
  • Press play on the song in the app — audio streams correctly
  • Check the Netlify deploy log (Site → Deploys) — no red error lines
If the upload fails: double-check your R2 env vars in Netlify (Site settings → Environment variables) — a single typo in R2_SECRET_ACCESS_KEY is the most common culprit. After fixing, trigger a new deploy.
