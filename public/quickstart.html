Deploy Your Music Streaming App

Free accounts. No terminal. About 20–30 minutes depending on your storage choice.

No terminal required Free tier options ~20–30 minutes
1
GitHub — copy the app to your account
GitHub is where the app lives. Netlify reads from it automatically.
Step 1

Create a free GitHub account

Go to github.com and click Sign up. The free plan is all you need.

Step 2

Fork the app repository

Go to the app's GitHub page and click Fork near the top-right, then Create fork. GitHub copies the entire app into your account.

2
Storage — choose one option
Complete all the steps inside your chosen option, then move to Phase 3.
Backblaze B2
Recommended No credit card No domain needed
Free to sign up. 10 GB free storage. No domain or credit card required. Very cheap beyond the free tier ($0.01/GB).
▼
A · 1

Create a free Backblaze account

Go to backblaze.com and sign up. No credit card is required.

A · 2

Create a storage bucket

In the dashboard, click B2 Cloud Storage in the sidebar, then Create a Bucket. Name it (e.g. my-music). Under Files in Bucket are select Public. Click Create a Bucket.

The bucket name must be globally unique across all Backblaze users — add a few random characters if your first choice is taken.
A · 3

Save your endpoint and friendly URL

Click on your new bucket. Find and copy two things from the bucket page:

Friendly URLhttps://f004.backblazeb2.com/file/my-music
The numbers (004 etc.) will differ for you — use whatever your dashboard shows.
A · 4

Create an application key

In the sidebar click Application Keys → Add a New Application Key. Give it any name. Under Allow access to Bucket(s) select your bucket. Set Type of Access to Read and Write. Click Create New Key.

You'll see two values. Copy both now — the application key is only shown once:

Cloudflare R2
Best performance Credit card required Domain required
Truly zero egress fees. Best global performance. Requires a domain on Cloudflare and a payment card on your account (you won't be charged within the free tier).
▼
B · 1

Create a free Cloudflare account

Go to cloudflare.com and sign up. You will need to add a payment card to activate R2 — even on the free tier. You won't be charged unless you exceed the free limits (10 GB storage, 10 million reads/month, unlimited data transfer).

B · 2

Add your domain to Cloudflare

In the dashboard click Add a domain. Enter your domain and choose the Free plan. Cloudflare will show you two nameserver addresses — copy them.

Log in to wherever you bought your domain and replace the existing nameservers with the two Cloudflare ones. Save the change. Cloudflare will email you when propagation is complete (usually minutes to a couple of hours).

Already using Cloudflare for your domain? Skip to B·3.
B · 3

Create a storage bucket

In the sidebar go to Storage & Databases → R2 Object Storage → Create bucket. Name it (e.g. music) and click Create bucket.

B · 4

Connect your domain to the bucket

Open the bucket → Settings tab → scroll to Public Access → Custom Domains → Connect Domain.

Enter a subdomain, e.g. cdn.yourdomain.com. Click Continue, review the record, then Connect Domain. Wait for the status to show Active.

Copy and save the full custom domain URL, e.g. https://cdn.yourdomain.com.

This step requires your domain to already be on Cloudflare's nameservers (step B·2). If it isn't active yet, wait and come back.
B · 5

Get your Account ID

Navigate back to R2 Object Storage in the sidebar. On the right find the Account Details box — copy your Account ID.

B · 6

Create an API token

In the Account Details box click Manage next to API Tokens → Create API token. Set permissions to Object Read & Write. Click Create API token.

Copy both values — the secret is only shown once:

FTP / Web Hosting
If you already have hosting Paid hosting required
Use your existing web host's file storage. Good if you already pay for shared hosting. No special storage account needed.
▼
C · 1

Get a web hosting account with FTP access

You need a standard shared hosting plan that gives you FTP access and a web-accessible file directory. Most shared hosts work. Hostinger is a reasonable choice — their shared hosting starts at a few dollars a month and includes FTP, a domain, and a straightforward control panel (hPanel).

If you already have a host with FTP access, skip ahead to C·2.

C · 2

Find your FTP credentials

Log into your host's control panel and look for an FTP Accounts or File Manager section. You need:

What you needWhere to find it
FTP hostnameUsually your domain name or something like ftp.yourdomain.com — shown in your host's FTP settings
FTP usernameYour FTP account username
FTP passwordYour FTP password
Hostinger users: go to hPanel → Files → FTP Accounts. Your FTP hostname is shown there. You can set an FTP password on that page if you haven't already.
C · 3

Note your public uploads URL

Audio files will be uploaded into a folder on your hosting (the app uses uploads/ by default, sitting inside your public web root). You need the public URL that corresponds to that folder.

For most hosts this is simply https://yourdomain.com/uploads. Check that this URL is accessible in a browser — if it shows a directory listing or a blank page rather than a 404, you're good.

The folder doesn't need to exist yet — the app creates it automatically on first upload.
C · 4

Check whether your host uses FTPS

Some hosts require a secure FTP connection (FTPS). Your host's FTP settings page will tell you. If it says Explicit TLS or FTPS, you'll need to set FTP_SECURE=true below.

Environment variables to add in Netlify (Step 5)
Name (type exactly)Value
FTP_HOSTYour FTP hostname, e.g. ftp.yourdomain.com
FTP_USERYour FTP username
FTP_PASSWORDYour FTP password
FTP_PUBLIC_BASE_URLThe public URL to your uploads folder, e.g. https://yourdomain.com/uploads
FTP_BASE_PATHThe path to your uploads folder on the server, e.g. public_html/uploads — ask your host if unsure
FTP_SECUREtrue if your host uses FTPS, otherwise false
FTP_BASE_PATH is the server-side folder path, not a URL. On Hostinger it is typically public_html/uploads. On other hosts it might just be uploads or www/uploads — check your host's file manager to confirm where your public web root is.
3
Netlify — publish your app
Netlify connects to GitHub and deploys automatically.
Step 3

Create a free Netlify account

Go to netlify.com and click Sign up. Choose Sign up with GitHub — this connects the two accounts in one click.

Step 4

Import your GitHub repository

Click Add new site → Import an existing project → Deploy with GitHub. Authorize Netlify, then select your forked Full-generic-music-streaming-app repository. Netlify reads netlify.toml automatically — don't change any build settings.

Step 5

Add your environment variables

Before clicking deploy, click Add environment variables. Add all the variables from the storage option you chose in Phase 2 — the table is at the bottom of that section.

Then add one more that applies to all options:

Name (type exactly)Value
ADMIN_API_TOKENMake up any long string of random characters — this is your admin password
Variable names are case-sensitive. A typo is the most common reason uploads fail.
Step 6

Deploy

Click Deploy site. Netlify builds and publishes your app in about a minute. When it finishes you'll see a green Published status and a live link like https://your-app-name.netlify.app.

That's it for setup. You manage everything — uploading music, editing settings — through the app's admin panel. You don't need to touch GitHub again.
4
Check everything works
Run through this before sharing your link
  • Open your Netlify URL — the app loads
  • Log in to the admin panel using the ADMIN_API_TOKEN you set in Step 5
  • Upload a song through the admin upload screen
  • Check your storage (Backblaze / Cloudflare / hosting file manager) and confirm the file is there
  • Press play — the audio streams without errors
If uploads fail: go to Netlify → Site configuration → Environment variables, double-check every value against the table in your chosen storage option, then go to Deploys and click Trigger deploy.
