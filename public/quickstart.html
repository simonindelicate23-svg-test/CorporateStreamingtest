Deploy Your Music Streaming App

Three free accounts. No coding. No terminal. About 20 minutes.

No terminal required Free tier friendly ~20 minutes
1
GitHub — copy the app to your account
GitHub is where the app lives. Netlify reads from it automatically.
Step 1

Create a free GitHub account

Go to github.com and click Sign up. Use any email address. The free plan is all you need.

Step 2

Fork the app repository

Go to the app's GitHub page and click the Fork button near the top-right of the page.

Leave all settings as they are and click Create fork. GitHub will copy the entire app into your account — no file uploads needed.

2
Cloudflare R2 — store your audio files
Free to stream — no charge when listeners play your music.
Step 4

Create a free Cloudflare account

Go to cloudflare.com and click Sign up. You don't need to add a website or a credit card.

Step 5

Create a storage bucket

In the Cloudflare dashboard, click R2 in the left sidebar, then click Create bucket.

Name it something like music-uploads. Leave everything else as-is and click Create bucket.

Step 6

Make it publicly accessible

Open your new bucket, click the Settings tab, scroll down to Public access, and click Allow Access.

Cloudflare will show you a public URL that looks like https://pub-xxxxxxxx.r2.dev — copy and save this. You'll need it in Step 11.

Step 7

Get your Account ID

Look at the right-hand sidebar on any R2 page. You'll see Account ID — copy and save it.

Step 8

Create an API token

On the R2 overview page, click Manage R2 API tokens in the top-right, then Create API token.

Set permissions to Object Read & Write. Click Create API token.

You'll see two values on the next screen. Copy both — you won't be able to see them again:

Label on screenSave it as
Access Key IDyour R2 Access Key
Secret Access Keyyour R2 Secret
Paste these into a notes app right now before clicking away. You cannot retrieve the secret again.
3
Netlify — publish your app to the internet
Netlify connects to GitHub and deploys automatically.
Step 9

Create a free Netlify account

Go to netlify.com and click Sign up. Choose Sign up with GitHub — this connects the two accounts in one click, no extra setup needed.

Step 10

Import your GitHub repository

In the Netlify dashboard click Add new site → Import an existing project → Deploy with GitHub.

Authorize Netlify when prompted, then select your forked Full-generic-music-streaming-app repository from the list. Netlify will read the netlify.toml file and configure everything automatically — don't change any build settings.

Step 11

Add your secret keys

Before clicking deploy, click Add environment variables. Add each of the following — use the values you saved in Steps 7 and 8:

Name (type exactly)Value
R2_ACCOUNT_IDYour Account ID from Step 7
R2_ACCESS_KEY_IDAccess Key ID from Step 8
R2_SECRET_ACCESS_KEYSecret Access Key from Step 8
R2_BUCKET_NAMEThe bucket name you chose, e.g. music-uploads
R2_PUBLIC_BASE_URLThe https://pub-xxx.r2.dev URL from Step 6
ADMIN_API_TOKENMake up any long string of random characters — this becomes your admin password
The variable names must be typed exactly as shown — they are case-sensitive. A typo here is the most common reason uploads fail.
Step 12

Deploy

Click Deploy site. Netlify will build and publish your app — this takes about a minute.

When it finishes you'll see a green Published status and a live link like https://your-app-name.netlify.app. That's your app.

That's it for setup. Once it's live, you manage everything — uploading music, editing settings — through the app's admin panel. You don't need to touch GitHub again.
4
Check everything works
Run through this before sharing your link
  • Open your Netlify URL — the app loads
  • Log in to the admin panel using the ADMIN_API_TOKEN you set in Step 11
  • Upload a song through the admin upload screen
  • Open your R2 bucket in Cloudflare and confirm the file appears
  • Press play — the audio streams without errors
If uploads fail: go to Netlify → Site configuration → Environment variables and double-check every value — especially R2_SECRET_ACCESS_KEY and R2_PUBLIC_BASE_URL. After fixing, go to Deploys and click Trigger deploy.
