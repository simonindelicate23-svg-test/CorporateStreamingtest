Import Catalogue

Migrate tracks and files from another deployment directly into this instance's R2 storage.

⚠️
Advanced Read before you proceed

This tool copies all tracks and audio files from another instance into this deployment. Files are transferred server-to-server (source URL → this R2 bucket) — nothing passes through your local machine or the source Netlify functions.

Duplicate tracks are detected by ID and skipped automatically, so re-running is safe. This instance must have R2 (or Backblaze B2) configured. The source instance must have both "Expose catalogue API" and "Allow catalogue importing" enabled in its Site Settings.

Once the import is complete, turn off "Allow catalogue importing" on the source instance to stop exposing audio URLs publicly.
