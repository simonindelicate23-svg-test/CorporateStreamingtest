Subscriptions & payment gating

When off, lock icons and paywall modals are hidden even if tracks are marked as paid.

Subscribers pay once a month and unlock all gated tracks. No database required — access is verified directly with PayPal and stored as a signed token in the listener's browser.

Create a billing plan in your PayPal Developer dashboard → Products & Plans. Copy the Plan ID here.

Display-only — shown in the paywall modal. Must match your PayPal plan price.

PAYPAL_API_BASE https://api-m.sandbox.paypal.com for testing  |  https://api-m.paypal.com for live.

Make your own player...

Build a custom music player for your catalogue using any AI assistant.

Want a player for your music with a bit more character? Now your site is up and running, you can expose your catalogue via the API toggle above and display your music on any site that knows how to fetch it. Build something however you like — go to any AI assistant (Claude, ChatGPT, or similar), paste the prompt below, and it'll have everything it needs to create a custom music player. Just keep this site running with the API exposed and go wild.

Build a single self-contained HTML file — no build tools, no frameworks, vanilla
JS and inline CSS only — that browses and plays a music catalogue from this API.

BEFORE YOU WRITE ANY CODE, ASK ME:
─────────────────────────────────────────────

"What visual style and feel would you like for this music player?"

─────────────────────────────────────────────
BASE URL
─────────────────────────────────────────────

At the top of the html file, prominently declare:

  const BASE_URL = 'https://YOUR-SITE-URL-HERE.netlify.app';

The user will replace this string with their own site URL before opening the file.
Add a comment making this obvious.

─────────────────────────────────────────────
API
─────────────────────────────────────────────

All endpoints are public — no auth, CORS open.

GET {BASE_URL}/catalogue

Returns everything needed in one call:

{
  "instance": {
    "name": string,
    "description": string,
    "logoUrl": string
  },
  "releases": [
    {
      "id": string,
      "title": string,
      "artist": string,
      "artworkUrl": string,
      "genre": string,
      "year": number,
      "trackCount": number,
      "tracks": [
        {
          "id": string,
          "title": string,
          "trackNumber": number,
          "durationSeconds": number,
          "audioUrl": string,     // direct mp3 URL — use as <audio src> for free tracks
          "gated": boolean,       // true = paid/locked content
          "artworkUrl": string,
          "genre": string,
          "year": number,
          "description": string,
          "medium": string        // "studio", "live", etc.
        }
      ]
    }
  ]
}

AUDIO PLAYBACK
  Free tracks (gated: false): set <audio src="${track.audioUrl}"> directly.
  Gated tracks (gated: true):  show a padlock, don't attempt playback.
  No payment UI needed — just mark them as locked visually.

SHARING
  Each track has a shareable page at: {BASE_URL}/s/{track.id}
  Include a copy-to-clipboard share button per track.

─────────────────────────────────────────────
FEATURES
─────────────────────────────────────────────

BROWSE
  - Album cards: artwork, title, artist, year, track count.
  - Click album to see its tracks and auto-select the first track
  - "All Tracks" flat view as alternative.
  - Search / filter by name, artist, genre.

PLAYER BAR (persistent, bottom of screen)
  - Artwork, track name, artist, album.
  - Prev / play-pause / next, seek bar, volume, elapsed/total time.
  - Playing an album queues its tracks in order.
  - Keyboard: Space = play/pause, ← / → = prev/next.

TRACK DETAIL (click a track)
  - Large artwork, liner notes (description field), genre, year, medium.
  - Share button.

STATES
  - Skeleton loaders while fetching.
  - Friendly error message + retry if the API call fails.
  - Responsive: works from 320 px to 1440 px+.

─────────────────────────────────────────────
CONSTRAINTS
─────────────────────────────────────────────

- Single .html file, all CSS and JS inline.
- Native <audio> element only.
- Use inline SVG for icons (no icon CDN dependency).
- Format durations as m:ss.
