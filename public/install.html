Launch your music site on Netlify with zero guesswork

This page focuses on the hosted flow only: deploy to Netlify, add MongoDB, and run the app in production. No local machine setup required.

Tip: keep this page open while you work through the steps.

1) Deploy this repository to Netlify

Start here

In Netlify, click Add new site → Import an existing project, select this repo, and deploy.

Use the default build settings from the repository unless you've already customized them.

Netlify dashboard → Add new site → Import an existing project → Deploy site
After deploy, open your Netlify site URL.

2) Open the hosted install page

Required

Go to your deployed site and open /install.html in the browser.

https://<your-netlify-site>/install.html

3) Node runtime is already handled

No local install

Yes — Node is already running in Netlify Functions for your deployed site. You do not need to install Node.js locally for this flow.

Just continue with cloud configuration below.

No local terminal commands needed for deployment setup.

4) Set up MongoDB (Atlas recommended)

Required

Create a MongoDB Atlas cluster (or use an existing one), then create a database user and allow Netlify access in Network Access.

Copy your connection string and database name, then paste values below to generate environment variables for Netlify.

MONGODB_URI=
MONGODB_DB_NAME=
APP_BASE_URL=https://your-site.netlify.app
Paste these into Netlify → Site configuration → Environment variables.

5) Redeploy and verify

Go live
Netlify dashboard → Trigger deploy → Deploy site
Then open /player.html on your live URL
Example: https://your-site.netlify.app/player.html.
